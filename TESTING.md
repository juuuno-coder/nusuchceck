## 🌐 브라우저 테스트

### 테스트 환경

#### 데스크톱 브라우저
- [ ] **Chrome** (최신 버전)
- [ ] **Firefox** (최신 버전)
- [ ] **Safari** (macOS, 최신 버전)
- [ ] **Edge** (최신 버전)

#### 모바일 브라우저
- [ ] **Safari** (iOS 15+)
- [ ] **Chrome** (Android 10+)
- [ ] **Samsung Internet** (Android)

### 테스트 체크리스트

#### 1. 기본 폼 동작
- [ ] 페이지 로딩이 3초 이내에 완료됨
- [ ] 프로그레스 바가 정상적으로 표시됨
- [ ] 각 단계 전환이 부드럽게 작동함 (페이드인 애니메이션)
- [ ] "다음" 버튼이 하단에 고정되어 표시됨 (모바일)

#### 2. Step 1: 증상 선택
- [ ] 6개의 증상 버튼이 그리드로 표시됨
- [ ] 버튼 클릭 시 파란색으로 변경되고 체크마크 표시
- [ ] 선택 후 400ms 후 자동으로 다음 단계로 이동
- [ ] 이전 선택 해제 시 체크마크가 사라짐

#### 3. Step 2: 건물 종류 선택
- [ ] 7개의 건물 버튼이 3열 그리드로 표시됨
- [ ] 버튼 선택 동작이 Step 1과 동일하게 작동
- [ ] 모바일에서 버튼이 적절한 크기로 표시됨 (터치 최적화)

#### 4. Step 3: 주소 입력
- [ ] "검색" 버튼 클릭 시 다음 주소 API 팝업 열림
- [ ] 주소 선택 후 필드에 자동 입력됨
- [ ] 상세 주소 입력 가능
- [ ] 필수 필드 미입력 시 "다음" 버튼 비활성화

#### 5. Step 5: 영상 업로드 ⭐ 중요
- [ ] **클릭 업로드**
  - [ ] 업로드 영역 클릭 시 파일 선택 대화상자 열림
  - [ ] 비디오 파일만 선택 가능 (MP4, MOV, WebM)
  - [ ] 여러 파일 선택 가능 (최대 10개)

- [ ] **드래그 앤 드롭** (데스크톱만)
  - [ ] 파일을 드래그하면 업로드 영역이 확대되고 색상 변경
  - [ ] 비디오 파일을 드롭하면 업로드됨
  - [ ] 이미지 파일 드롭 시 "영상 파일만..." alert 표시

- [ ] **미리보기**
  - [ ] 선택한 영상이 카드 형태로 표시됨
  - [ ] 파일명, 크기, 타입이 정확하게 표시됨
  - [ ] 비디오 플레이어가 작동함 (controls)
  - [ ] X 버튼으로 개별 영상 제거 가능

- [ ] **대용량 파일 경고**
  - [ ] 100MB 이상 파일 선택 시 노란색 경고 박스 표시
  - [ ] 총 파일 크기가 정확하게 표시됨

#### 6. Step 7: 최종 확인
- [ ] 선택한 증상, 건물, 주소가 요약에 표시됨
- [ ] "수정" 버튼 클릭 시 해당 단계로 이동
- [ ] 안심 보증 카드가 표시됨

#### 7. 제출
- [ ] "전문가에게 요청하기" 버튼 클릭 시 로딩 스피너 표시
- [ ] 로딩 텍스트: "영상 업로드 중... 잠시만 기다려주세요"
- [ ] 제출 완료 후 적절한 페이지로 이동

### 브라우저별 특이사항

#### Chrome
- [ ] Service Worker 정상 작동
- [ ] DevTools 콘솔에 에러 없음
- [ ] Lighthouse 점수 확인 (Performance > 85)

#### Safari
- [ ] 비디오 파일 업로드 정상 작동
- [ ] iOS에서 `capture="environment"` 속성 작동 확인
- [ ] 드래그 앤 드롭 작동 (macOS only)

#### Firefox
- [ ] DataTransfer API 정상 작동
- [ ] 파일 미리보기 정상 표시
- [ ] 애니메이션 부드럽게 작동

- [ ] 미리보기에서 영상 재생 가능

#### 화면 회전

## ♿ 접근성 테스트

### 스크린 리더 테스트

#### NVDA (Windows)
1. NVDA 실행 (`Ctrl + Alt + N`)
2. 폼 페이지 접속
3. 테스트 항목:
   - [ ] 프로그레스 바가 "폼 작성 진행률 14%" 등으로 읽힘
   - [ ] 버튼 선택 시 "벽면 누수 선택됨" 등으로 읽힘
   - [ ] 영상 업로드 영역이 "영상 파일 업로드: 촬영하거나..." 등으로 읽힘
   - [ ] 파일 선택 시 "3개 영상 선택됨" 등으로 읽힘

#### JAWS (Windows)
- [ ] 랜드마크 네비게이션 작동 (`R` 키)
- [ ] 폼 모드 전환 정상 작동 (`Enter` 키)
- [ ] 헤딩 네비게이션 작동 (`H` 키)

#### VoiceOver (macOS/iOS)
1. VoiceOver 실행 (`Cmd + F5`)
2. 테스트 항목:
   - [ ] 로터로 폼 컨트롤 접근 가능
   - [ ] 제스처로 단계 이동 가능
   - [ ] 영상 미리보기 카드가 읽힘

### 키보드 네비게이션

#### Tab 네비게이션
- [ ] `Tab` 키로 모든 인터랙티브 요소 접근 가능
- [ ] 포커스 표시가 명확함 (파란색 링)
- [ ] `Shift + Tab`으로 역순 이동 가능

#### 키보드 단축키
- [ ] `Enter` / `Space`: 버튼 선택
- [ ] `Arrow Keys`: 버튼 그룹 내 이동
- [ ] `Escape`: 폼 닫기 확인 대화상자
- [ ] 영상 업로드 영역에서 `Enter` / `Space`: 파일 선택

#### 포커스 관리
- [ ] 단계 전환 시 자동으로 첫 번째 요소에 포커스
- [ ] 모달 열림 시 포커스 트랩
- [ ] 에러 발생 시 에러 메시지로 포커스 이동

### 색상 대비
- [ ] 모든 텍스트가 WCAG AA 기준 충족 (4.5:1)
- [ ] 비활성화된 버튼도 충분한 대비 (3:1)
- [ ] 링크 색상이 배경과 4.5:1 대비

### 확대/축소
- [ ] 200% 확대 시 레이아웃 깨지지 않음
- [ ] 400% 확대 시에도 사용 가능
- [ ] 텍스트 크기 변경 시 레이아웃 유지

## ⚡ 성능 테스트

### Lighthouse 점수

Chrome DevTools > Lighthouse 실행:

- [ ] **Performance**: 85점 이상
- [ ] **Accessibility**: 95점 이상
- [ ] **Best Practices**: 90점 이상
- [ ] **SEO**: 85점 이상

### Core Web Vitals

- [ ] **LCP** (Largest Contentful Paint): < 2.5초
- [ ] **FID** (First Input Delay): < 100ms
- [ ] **CLS** (Cumulative Layout Shift): < 0.1

### 네트워크 성능

#### Slow 3G 시뮬레이션
1. Chrome DevTools > Network > Slow 3G
2. 테스트:
   - [ ] 페이지 로딩 10초 이내
   - [ ] 로딩 인디케이터 표시
   - [ ] 필수 기능 사용 가능

#### 파일 크기
- [ ] JavaScript 번들 < 200KB (gzipped)
- [ ] CSS < 50KB (gzipped)
- [ ] 총 페이지 크기 < 500KB (초기 로드)

## 🐛 이슈 리포팅

### 버그 발견 시

GitHub Issue 생성 또는 다음 템플릿 사용:

```markdown
## 버그 제목
[브라우저] 간단한 설명

## 환경
- 브라우저: Chrome 120.0.6099
- OS: Windows 11
- 디바이스: Desktop

## 재현 단계
1. 폼 페이지 접속
2. Step 5로 이동
3. 영상 파일 드래그 앤 드롭
4. ...

## 예상 동작
영상이 업로드되고 미리보기가 표시되어야 함

## 실제 동작
alert "영상 파일만..." 표시됨

## 스크린샷
[스크린샷 첨부]

## 추가 정보
- 파일 크기: 150MB
- 파일 형식: MP4
- 네트워크: WiFi
```

### 심각도 분류

- **Critical**: 폼 제출 불가, 페이지 크래시
- **High**: 주요 기능 작동 안 함 (영상 업로드 실패)
- **Medium**: 부분적 기능 이상 (미리보기 안 보임)
- **Low**: UI 깨짐, 애니메이션 끊김
